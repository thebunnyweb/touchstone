# touchstone

**The LLM judge that corrects for its own biases.**

```bash
pip install touchstone-llm[anthropic]
```

```python
from touchstone import Judge, HELPFULNESS

judge = Judge(model="claude-sonnet-4-6")
result = await judge.score(output, rubric=HELPFULNESS)

print(result.adjusted_score)   # bias-corrected score
print(result.bias_flags)       # ["verbosity"] if flagged
print(result.passed)           # True / False vs threshold
```

## Why touchstone?

Every LLM eval framework runs a judge prompt and returns a score. Nobody corrects for what happens *before* that score reaches you.

LLM judges carry three systematic biases that silently inflate or deflect scores:

| Bias | What it does | How touchstone fixes it |
|------|-------------|------------------------|
| **Verbosity** | Longer outputs score higher regardless of quality | Length-normalized penalty applied automatically |
| **Position** | In A/B comparisons, whichever answer comes first wins more often | Runs both orderings, reconciles with position-bias detection |
| **Self-preference** | A model scores outputs from the same model family higher | Detected and flagged on every call |

On top of that, **LLM judges drift**. Models get silently updated. touchstone lets you baseline a judge and alert when its scoring behavior shifts.

## What touchstone is not

touchstone is a **library primitive**, not a platform. It has no dashboard, no dataset storage, no tracing, no spans. If you want a full observability platform, use LangSmith. touchstone is what you call inside your evaluation logic.
